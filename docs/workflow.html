Cerafica Posting Pipeline

From camera capture to published Instagram post — the complete system

1 The Full Pipeline

Capture you

Take a photo or video of your work. Add it to the "To Post" album in the macOS Photos app, or drop framed files into output/framed/.

Run ./post script

This triggers the entire pipeline. One command generates everything.

./post   all framed videos
./post 3   just 3 videos
./post --no-ai   skip AI (fast, generic)

Export from Photos app AppleScript

In default mode, the script exports the original file from "To Post" via AppleScript. In --from-output mode (the default for ./post), it reads directly from output/framed/.

AI Vision Analysis Ollama / OpenRouter

The photo or video frames are sent to an AI vision model (Kimi K2.5 locally, or Gemini Flash via OpenRouter). It detects: piece type, colors, glaze, technique, mood, surface qualities, firing state.

Worldbuilding Lookup feedback.db

The filename is checked against the series_pieces table. If it matches a known photo or contains a planet name, the full planetary lore is loaded — surface geology, orbital data, formation history, inhabitants.

See Section 3 for the exact matching logic

Caption Generation AI writing model

The analysis + worldbuilding + voice rules + brand identity + geological vocabulary are all injected into an AI writing prompt. The AI generates 3 caption options. The best one is selected.

Worldbuilding pieces get planetary explorer voice.
Regular pieces get potter voice with few-shot examples.

Validation built-in

Caption is checked against: banned AI words, em-dash count (max 1), length (300-500 chars sweet spot), question presence, and food/fabric metaphors (for worldbuilding pieces).

Schedule Calculation algorithm

Optimal posting times are calculated based on engagement research. Posts are spread across the week with minimum 30-minute gaps. Priority slots: Tue-Thu 8AM, Mon/Fri 9AM.

Posting Pack output folder

A folder opens in Finder. Each subfolder contains the media file, a caption.txt to copy-paste, and a README.txt with schedule time and instructions.

Location: output/posting-pack-YYYY-MM-DD/

Manual Upload you

Open Meta Business Suite. For each folder: drag in the media, paste the caption, set the schedule time. Click Schedule. Done.

2 Two Modes of Operation

Default Mode (from Photos app)

Full pipeline including framing. For new photos that haven't been framed yet.

python scripts/auto-post.py
Photos app → Export → Frame → Analyze → Caption → Pack

From-Output Mode (pre-framed)

Skips Photos app and framing. For already-framed videos and photos. This is what ./post uses.

./post 3
output/framed/ → Analyze → Caption → Pack
Where to put files for from-output mode:
Videos: output/framed/video/ (e.g., Ceruleix-2_rotating.mp4)
Photos: output/framed/YYYY-MM-DD/ (e.g., IMG_4908_planetary.jpg)
3 How Files Get Matched to Planets

The connection between a physical photo/video and its planetary identity lives in a single SQLite database: human-door/feedback.db → series_pieces table.

Someone (you, or the worldbuilding pipeline) manually registered each photo filename alongside its planet name and lore. The database row is the source of truth — the filename doesn't encode the planet in any way.

How the Matching Algorithm Works

When a file enters the pipeline, lookup_worldbuilding(filename) runs two checks against the database:

# Input: "Ceruleix-2_rotating.mp4" # Strategy 1: Planet-name extraction # Regex: ^([A-Z][a-zA-Z]+(?:-[a-z]+)*-\d+) # Matches: "Ceruleix-2" from "Ceruleix-2_rotating" filename = "Ceruleix-2_rotating.mp4" → stem = "Ceruleix-2_rotating" → regex extracts "Ceruleix-2" → SQL: WHERE planet_name = 'Ceruleix-2' → MATCH FOUND # Input: "IMG_4908.JPG" # Strategy 1: Planet-name extraction filename = "IMG_4908.JPG" → stem = "IMG_4908" → regex: no match (starts with "IMG", not a capital letter + word) # Strategy 2: Photo filename match → SQL: WHERE LOWER(photo) = 'img_4908.jpg' → MATCH FOUND → planet_name = "Ceruleix-2" # Input: "random_photo.jpg" filename = "random_photo.jpg" → Strategy 1: no match → Strategy 2: no match → NO MATCH → worldbuilding = None (regular pottery caption)
The filename doesn't inherently contain the planet name. The connection is the database row. The planet-name regex is just a convenience for video files that were named after their planet (like Ceruleix-2_rotating.mp4). Original camera photos like IMG_4908.JPG rely entirely on the database lookup.

What Gets Loaded When a Match is Found

WorldbuildingData

  • planet_name — e.g., "Ceruleix-2"
  • series_name — "Glaze Exploration Series"
  • series_description — "Each vessel is a planet..."
  • surface_geology — geological description
  • orbital_data — atmosphere, temp, scent
  • formation_history — captain's log lore
  • inhabitants — what it feels like there
  • generated_caption — previously generated caption

What Changes in the AI Prompt

  • PLANETARY WORLDBUILDING block injected with all lore
  • GEOLOGICAL VOCABULARY block with color/surface translations
  • Voice shifts to planetary explorer language
  • Few-shot examples removed (replaced by lore)
  • Banned words expanded — food/fabric metaphors flagged
  • CTA changes — "Would you explore this world?"
