DeepSeek OCR & Book Digitization

TL;DR: DeepSeek OCR launched October 2025, v2 dropped January 2026. It's open source, runs locally, and works well. For your pottery books, start with PaddleOCR-VL (free, local) or GPT-4o (cloud, ~$0.02/page). Photo quality matters more than model choice.

What Happened with DeepSeek OCR

DeepSeek OCR was officially released October 17, 2025, with DeepSeek-OCR 2 following on January 27, 2026. It's not vaporware — it's a fully functional, open-source model on GitHub (22,700+ stars, MIT License).

How to Use DeepSeek OCR

▼

Local (free):

# Via Transformers
from transformers import AutoModel, AutoTokenizer
model_name = 'deepseek-ai/DeepSeek-OCR-2'
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
model = AutoModel.from_pretrained(model_name, trust_remote_code=True)
model = model.eval().cuda()
result = model.chat(tokenizer, 'page.jpg', ocr_type='format')

Cloud APIs:

  • Clarifai — API with OpenAI-compatible endpoint
  • Google Cloud Vertex AI — Available as MaaS
  • vLLM — Officially supported for efficient inference
  • Ollama — Available as deepseek-ocr model

Bonus tool: bookdatamaker on PyPI — a CLI specifically for extracting text from books using DeepSeek-OCR.

What Makes It Special — Architecture

▼

DeepSeek OCR uses a novel "visual causal flow" approach. Instead of scanning left-to-right like traditional OCR, it reorders visual tokens based on image semantics — how humans actually read. This lets it use just 1,120 visual tokens compared to 6,000+ for competitors like Qwen2.5-VL.

Performance claims:

  • 97% exact-match accuracy on Fox benchmark at 10x compression
  • 200k pages/day on single NVIDIA A100 GPU
  • 100+ languages supported
  • 57-86% reduction in Character Error Rate after fine-tuning

Recommended Approach for Your Pottery Books

The biggest lever isn't the OCR model — it's photo quality. Two lights at 45-degree angles, flat pages, phone camera is fine. Scanning apps like vFlat or Adobe Scan handle auto-crop and perspective correction.

Option 1: Start Simple (Free, Local)

PaddleOCR-VL — 94.5% accuracy on OmniDocBench, handles complex layouts, runs locally.

pip install paddlepaddle-gpu paddleocr
from paddleocr import PaddleOCR
ocr = PaddleOCR(use_angle_cls=True, lang='en')
result = ocr.ocr('page.jpg', cls=True)

Option 2: Best Quality (Paid Cloud)

GPT-4o — Best accuracy, can preserve layout and mark image locations. ~$0.01-0.05/page.

Option 3: Local Power User (Free, GPU Required)

DeepSeek-OCR 2 — State-of-the-art open source. NVIDIA GPU with 8GB+ VRAM recommended.

Scanning Tips

  • Lighting: Two strong lights at 45-degree angles (left + right). Eliminates shadows.
  • Camera: Smartphone 12-50MP is sufficient. Keep parallel to page.
  • Book support: Book cradle or stand at 90-degree angle minimizes page curl.
  • Apps: vFlat Scan (free, two pages at once), Adobe Scan, Microsoft Lens.

Claude Code UI/UX Guide

TL;DR: The status line at the bottom is your dashboard. Spinner = thinking. Static = idle. Ctrl+O toggles verbose mode (often subtle/bugged). Shift+Tab cycles permission modes. Esc interrupts generation.

The Status Line (Bottom of Screen)

This is your main dashboard. It shows:

  • Model — Opus / Sonnet / Haiku
  • Directory — where you are
  • Context bar (███░░░░) — token usage. Green <50%, yellow 50-80%, red >80%
  • Cost — session spend (if configured)
  • Git branch — current branch

Customize it: /statusline show model, context %, git branch

Activity States

What You SeeWhat It Means
Spinner cycling ⠋⠙⠹⠸ Claude is thinking/processing
[ToolName] description Executing a tool (Read, Bash, Edit...)
Static text, no animation Idle — waiting for your input
Allow? (y/n/a/e/d >) Permission prompt — needs approval
Gray italic text Extended thinking (when verbose mode works)

Permission Prompts

The key legend when you see Allow? (y/n/a/e/d >):

Permission modes cycle with Shift+Tab: Normal → Auto-accept edits → Plan mode → Bypass

Tool calls show in a hierarchy:

Verbose Mode (Ctrl+O)

Known issue: Verbose mode is often very subtle and has been bugged in some v2.1.x versions (GitHub issues #22977, #25980). Even when working, the main visible difference is gray italic thinking text and slightly more detailed tool info. It's not a dramatic UI change.

To verify it's on: run /config and check "Verbose output" shows true.

Alternative: add { "outputStyle": "verbose" } to ~/.claude/settings.json or start with --verbose flag.

Reducing Motion

If animations bother you, add to ~/.claude/settings.json:

{ "prefersReducedMotion": true }

Suppresses shimmer and spinner cycling. Note: blinking dot may still animate (known issue #29323).

Overcoming Sales Fear as a Neurodivergent Artist

Core insight: Sales systems were designed by and for neurotypical people. That's why they feel hard — not because something's wrong with you. Rejection Sensitive Dysphoria (RSD) is biological. Your brain processes rejection as physical pain. Knowing that creates distance: "That's my RSD talking," not "I'm inadequate."

Your Neurodivergent Traits Are Sales Superpowers

TraitHow It Helps Sales
HyperfocusDeep product knowledge that builds authentic trust
Pattern recognitionUnderstanding customer needs others miss
CreativityNovel ways to meet customer needs
AuthenticityStands out from polished marketing BS
Passion for craftGenuine excitement that sells naturally

Energy Management

  • Batch DMs: Check at set times (e.g., 10am, 3pm, 7pm). Turn off notifications outside those windows. Constant context-switching drains neurodivergent brains faster.
  • 3-Message Rule: Max 3 back-and-forths per conversation per day. If more depth needed, suggest continuing tomorrow.
  • Track your patterns: After 2 weeks, you'll know "I can handle 3 DM conversations on Tuesday but only 1 on Friday." Plan accordingly.
  • Schedule recovery time: After a Sunday shop update, plan Monday as studio-only day (no customer communication).

I can hold it for you for [X days]! If that works for you, just confirm and I'll mark it as reserved. After that timeframe, I'd need to release it back to available. Does that feel fair?

Instagram-Specific Tips

  • Use Stories, not live video — Stories can be pre-recorded when you have energy. Live video requires real-time social processing (exhausting).
  • Create a "DM FAQ" Story Highlight — Pricing range, shipping, payment methods, response time. Reduces repetitive questions before they become DMs.
  • Comment-to-Claim system — Post pieces numbered. "Comment the number to claim, I'll DM you payment details." Public record, clear system, you process in batch.
  • Pre-write shop update announcements — Write once when you have energy, copy-paste for each drop.
  • Set boundaries in your bio — "DMs open. Responses within 24hrs. Shop updates: Sundays @ 4pm PST."

Gradual Exposure (Your Pace, No Timelines)

TimelineGoalActions
Weeks 1-2 Low-stakes interaction Reply to comments with emojis. Like other ceramicists' posts. Post a non-sales story question.
Weeks 3-4 Structured sales practice Create DM FAQ highlight. Pre-write shop update template. Post ONE piece using comment-to-claim.
Weeks 5-6 Active sales DMs Post 3-5 pieces. Use pre-written templates. Set response time expectations. Limit 3 DMs/day.

If a step triggers intense anxiety, go back one step. There's no deadline. Celebrate completing the action, not the sales outcome.

Instagram DM Automation (Without Getting Banned)

TL;DR: Instagram DM automation is safe when using Meta-approved tools (ManyChat, Inro, Spur) connected via OAuth. Never give your password to any tool. Start with native Quick Replies + FAQ buttons (free, zero risk), then add ManyChat free tier for comment-to-DM flows.

DO vs DON'T

Safe (DO)

  • Use official API tools (OAuth login)
  • Respond within 24h to user-initiated interactions
  • Send up to 200 DMs/hour via official API
  • Use Quick Replies / Saved Replies
  • Automate comment-to-DM flows
  • Set up FAQ auto-responses
  • Use away messages outside business hours

Banned (DON'T)

  • Never give your Instagram password to any tool
  • No cold DM outreach (unsolicited)
  • No browser bots or Chrome extensions
  • No mass following/unfollowing automation
  • No scraping usernames for bulk DMs
  • Don't message based on passive actions (likes alone)
  • Avoid repetitive identical messages

Rate Limits

MethodSafe Volume
Manual (new account <30 days)50-150 DMs/day
Manual (established account)100-150 DMs/day
Official API200 DMs/hour hard cap (150-180 safe)
The 24-Hour Messaging Window: You can only send automated DMs to users who engaged with you in the last 24 hours (they DM'd you, commented, replied to your Story, mentioned you, or clicked a Click-to-DM ad). Every time they reply, the clock resets.

Native Features to Use First (Free)

  • Saved Replies / Quick Replies — Settings → Business tools → Saved replies. Type /ship and it expands to your full shipping policy.
  • Away Messages — Auto-replies when unavailable.
  • FAQ Buttons — Up to 4 tappable buttons on your profile for instant answers.
  • Comment Keywords — Set auto-DMs when people comment specific words (e.g., "PRICE" → auto-DM pricing).
