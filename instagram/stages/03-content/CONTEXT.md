# Stage: Content -- Captions, Hashtags, CTAs

Write all copy for the week's posts.

## Inputs

| Source | What | Location |
|--------|------|----------|
| Content calendar | What to write for | `../02-planning/output/` |
| Voice rules | How to sound | `../../brand/voice-rules.md` |
| Caption templates | Patterns to follow | `references/caption-templates.md` |
| Hashtag library | Hashtag sets | `../../shared/hashtag-library.md` |
| CTA patterns | Sales/engagement CTAs | `references/cta-patterns.md` |

## Process

1. For each post in calendar, generate caption:
   - Match pillar type (finished work vs. process vs. studio)
   - Follow voice rules
   - Include context/story from asset notes
2. Add hashtags:
   - Mix of broad + niche + branded
   - Vary sets between posts
3. Add CTA:
   - Sales posts: "DM to claim" / claim instructions
   - Process posts: engagement question
   - Studio posts: conversational prompt
4. Run audit (see below)

## Output

| File | Location |
|------|----------|
| Caption pack | `output/captions-[date].md` |

Format: Day-by-day with pillar, asset, caption, hashtags, and CTA for each post.

## Audit Checklist

Before finalizing, verify:

| Check | Pass Condition |
|-------|---------------|
| Voice match | Sounds like artist/maker, not corporate |
| CTA clarity | Clear next action for each post |
| Hashtag variety | Not identical sets on every post |
| Sales balance | Not every post is a sales post |
| Length variety | Mix of short and medium captions |
