# Stage: Planning -- Weekly Content Calendar

Decide what to post when, aligned with goals.

## Inputs

| Source | What | Location |
|--------|------|----------|
| This week's assets | Photos/videos to work with | `../01-input/output/` |
| Content pillars | Strategy framework | `references/content-pillars.md` |
| Brand identity | Voice/positioning | `../../brand/identity.md` |
| Previous calendar | Continuity | `output/` (last week) |

## Process

1. Review assets from `01-input`
2. Categorize by content pillar:
   - Finished work (sales posts)
   - Process/making
   - Studio life
   - Atmospheric/mood
3. Map to posting schedule:
   - How many feed posts this week?
   - Which days?
4. Balance goals:
   - Sales posts (show available work)
   - Process/atmospheric (build connection)
5. Create day-by-day calendar
6. Run checkpoint: does this week hit the right balance?

## Output

| File | Location |
|------|----------|
| Content calendar | `output/content-calendar-[date].md` |

### Output Format

```markdown
# Content Calendar: [Date Range]

## This Week's Balance
- Feed posts: [X]
- Sales-focused: [X] | Process/atmospheric: [X]

## Daily Plan

### [Day 1]
| Type | Asset | Pillar | Notes |
|------|-------|--------|-------|
| Feed | [piece name] | finished work | available, DM to claim |

### [Day 2]
| Type | Asset | Pillar | Notes |
|------|-------|--------|-------|
| Feed | [process shot] | behind-the-scenes | throwing video |

[Continue for each posting day]

## Pending for Next Week
- [pieces not scheduled this week]
```

## Checkpoint

Present calendar and ask:
- "Does this balance feel right? More sales push or more process content?"

## Next Stage

When calendar is confirmed, continue to `stages/03-content/CONTEXT.md`
